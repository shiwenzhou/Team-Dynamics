# Prompt Log: Bio-Behavioral Team Dynamics Analytics Chatbot Prototype

This prompt log documents self-generated prompts used to develop the synthetic-data Streamlit prototype. The prompts were not copied from the chat window. They are written as reproducible development prompts that could be reused with ChatGPT or Claude for code generation, debugging, and validation.

## Prompt 1: Scope the prototype around the dissertation AI bot

**Prompt:**

> You are a research software assistant. Help design a small Streamlit prototype for a dissertation-support chatbot. The app should use synthetic data only. Its purpose is to help users understand bio-behavioral team dynamics analytics: interdependence, adaptation, and influence distribution. Do not frame the app as testing dissertation hypotheses. Design the app as supplementary technical material for the AI-bot interpretation portion of the dissertation.

**Use in workflow:** Defined the app purpose and clarified that the public prototype should not use restricted data or claim dissertation findings.

**Validation:** Checked the proposal language and kept the prototype centered on interpretation of bio-behavioral team analytics rather than empirical hypothesis testing.

## Prompt 2: Generate safe synthetic time-series data

**Prompt:**

> Create a Python function that generates a synthetic 1 Hz time series for a five-role Fire Support Team-style demonstration. Include placeholder role-level features for heart rate, respiration, EEG alpha power, gaze-task focus, communication activity, symbolic role states, symbolic team state, phase label, and one synthetic event marker. Make the data realistic enough to test plots and metrics, but clearly label it as synthetic and not as BioTDMS output.

**Use in workflow:** Produced the public synthetic dataset structure used by the Dashboard, Synthetic data tab, Chatbot, and notebook.

**Validation:** Verified that the dataframe has the expected columns, one row per second, no personally identifying information, no raw physiological records, and a synthetic event marker within the scenario duration.

## Prompt 3: Implement metric functions

**Prompt:**

> Implement modular Python functions for moving-window Shannon entropy, moving-window categorical sample entropy, inverse sample entropy, normalized mutual information, AMI share, and HHI concentration. The functions should accept a dataframe generated by the synthetic-data function and return pandas dataframes ready for Streamlit tables and Matplotlib plots.

**Use in workflow:** Built the local Python analysis pipeline.

**Validation:** Checked that entropy values are nonnegative, normalized entropy is within 0 to 1, inverse sample entropy is not entirely missing, AMI shares sum to 1 within each moving window, and required output files are created.

## Prompt 4: Build Streamlit UI with connected session state

**Prompt:**

> Build a Streamlit app with the tab order Dashboard, Synthetic data, Chatbot. The Synthetic data tab must generate or update the dataframe and immediately rerun the analyses. The Dashboard and Chatbot must both read from the same st.session_state analysis bundle. Add a dataset ID so users can confirm that a changed synthetic dataset is the one being used by the chatbot.

**Use in workflow:** Reorganized the UI and restored the requested tab order.

**Validation:** Inspected the app logic to confirm that `generate_data_and_run_analysis()` writes `timeseries_df`, `analysis_bundle`, `settings`, and `dataset_id` to `st.session_state`, and that both Dashboard and Chatbot read from `st.session_state.analysis_bundle`.

## Prompt 5: Fix stale figures and tables in chat responses

**Prompt:**

> The Streamlit chatbot keeps showing the same figures and tables after the synthetic dataset changes. Refactor the app so chat responses store the figure bytes and table dataframe snapshots generated for that response. Do not render chat artifacts from static file paths that may be overwritten or cached.

**Use in workflow:** Changed the chatbot artifact renderer so each assistant message contains figure bytes and table snapshots tied to a dataset ID.

**Validation:** Confirmed that message artifacts include `dataset_id`, image bytes, and dataframe snapshots. This prevents earlier messages from silently changing when a later dataset overwrites output PNG and CSV files.

## Prompt 6: Connect chatbot requests to local Python analyses

**Prompt:**

> Add request parsing so the chatbot can respond to natural-language commands such as 'run all analyses and show figures and tables,' 'show the AMI table,' 'explain the entropy peak,' or 'generate a new synthetic dataset with seed 7, 1200 seconds, event 600.' Python should run the analyses before any LLM explanation. Gemini should only explain the current computed outputs.

**Use in workflow:** Added local request classification, optional data regeneration from chat commands, local metric recomputation, current figure/table selection, and Gemini prompt construction.

**Validation:** Reviewed that Gemini receives a JSON packet containing already-computed summaries, dataset ID, requested figures, requested tables, and a clear instruction not to invent calculations or evaluate individuals.
